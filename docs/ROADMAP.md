# BlockType 开发路线图

## 项目概述

BlockType 是一个使用 AI 辅助开发的 C++26 编译器，支持中英双语编程。

## 开发阶段

### Phase 0: 项目基础设施 (2 周) ✅ 进行中

**目标**: 建立完整的构建系统、项目骨架和测试框架

**主要任务**:
- [x] CMake 构建系统配置
- [x] LLVM 依赖集成
- [x] 项目目录结构
- [x] 基础类型定义 (SourceLocation, Diagnostics)
- [x] 国际化基础设施
- [x] AI 依赖配置
- [x] GTest 测试框架集成
- [ ] Lit 测试框架配置
- [ ] GitHub Actions CI/CD
- [ ] 文档完善

**验收标准**: 项目能成功编译，测试框架运行正常，CI/CD 流水线配置完成

---

### Phase 0.5: 双语支持设计 (1 周)

**目标**: 完成中英双语编程的详细设计和原型验证

**主要任务**:
- 设计双语关键字映射机制
- 设计双语标识符支持方案
- 设计诊断消息国际化框架
- 创建双语代码示例和测试用例

**验收标准**: 完成详细设计文档，原型验证通过

---

### Phase 0.6: AI 基础设施 (2 周)

**目标**: 建立多模型 AI 编排系统

**主要任务**:
- 实现 AI 接口抽象层
- 集成 OpenAI、Claude、Qwen、Ollama 等模型
- 实现智能路由和成本控制
- 建立缓存机制

**验收标准**: AI 系统可正常调用，支持多模型切换

---

### Phase 1: 词法分析器与预处理器 (3 周)

**目标**: 实现完整的词法分析和预处理功能

**主要任务**:
- Lexer 核心实现
- 双语关键字和标识符支持
- 预处理器指令处理
- Unicode 支持

**验收标准**: 能正确词法分析 C++26 代码（包括中文标识符）

---

**验收标准**: 能正确解析所有 C++26 表达式

### Phase 3: 声明解析 (5 周)

### Phase 4: 语义分析基础 (6 周)

### Phase 4.5: 可视化系统 (2 周)

### Phase 7.5: AI 集成 (3 周)

**目标**: 深度集成 AI 功能

- 智能代码补全

**验收标准**: AI 功能可用且有效

---

### Phase 8: 目标平台支持 (4 周)

**目标**: 实现多平台代码生成

**主要任务**:

- 集成测试
