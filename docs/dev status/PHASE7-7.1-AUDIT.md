Expr.h:1552 注释：`bool IsDirectInit; // true = auto{expr}, false = auto(expr)`

但 ParseExprCXX.cpp:796-801 实际实现：
```cpp
if (Tok.is(TokenKind::l_paren)) {
  IsDirectInit = true;    // ❌ auto(expr) 被标记为 direct-init
} else if (Tok.is(TokenKind::l_brace)) {
  IsDirectInit = false;   // ❌ auto{expr} 被标记为非 direct-init

```

根据 C++ 标准：
- `auto{expr}` 是 **direct-initialization**（花括号直接初始化）

**Parser 实现与 AST 注释及 C++ 标准语义均相反。**

**影响：** 如果 CodeGen 层根据 `IsDirectInit` 选择不同的构造策略（直接构造 vs 复制构造），将产生错误的行为。

### 1. ❌ SemaCXX 方法与 Parser 重复验证但未集成

**位置：** `src/Parse/ParseClass.cpp:726-731` vs `src/Sema/SemaCXX.cpp:24-80`

**建议：** 在 `Sema::ActOnFunctionDecl` 或 `Sema::ActOnCXXMethodDecl` 中调用 `SemaCXX::CheckExplicitObjectParameter`，并移除 Parser 中的重复检查。
