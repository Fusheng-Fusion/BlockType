## 一、项目总体目标和愿景

### 1.1 项目定位

BlockType 是一个**从零开发的 C++26 标准编译器前端**，后端使用 LLVM，核心特色：

| 特色 | 说明 |
|------|------|
| **C++26 标准** | 支持 C++26 反射、Contracts、#embed 等新特性 |
| **中英双语编程** | 原生支持中文和英文关键字、标识符、诊断消息 |
| **AI 驱动** | 深度集成 AI 能力，提供智能诊断和优化建议 |
| **LLVM 后端** | 使用 LLVM 17+ 生成高质量目标代码 |

### 1.2 目标平台

| 平台 | 架构 | 优先级 |
|------|------|--------|
| Linux | x86_64 | P0 |
| macOS | ARM64 (Apple Silicon) | P0 |
| Windows | x64 | P2（后续版本） |

### 1.3 项目愿景

1. **编译器层面**：成为首个完整支持 C++26 标准的独立编译器前端
2. **语言层面**：通过中英双语支持降低中文用户的编程门槛
3. **工具层面**：通过 AI 集成提供超越传统编译器的开发体验
4. **生态层面**：未来支持 WebAssembly、LSP 服务器、包管理器、增量编译

### 1.4 技术选型

| 技术 | 选择 | 理由 |
|------|------|------|
| 前端语言 | C++20 | 编译器自身实现语言 |
| 后端 | LLVM 17+ | 成熟的优化和代码生成基础设施 |
| 构建系统 | CMake | C++ 项目标准构建工具 |
| 测试框架 | Google Test + LLVM Lit | 单元测试 + 端到端测试 |
| Unicode | ICU / utf8proc | 中英双语和 Unicode 标识符支持 |
| AI 集成 | libcurl + nlohmann/json | 多模型 HTTP 调用 |
| 名称修饰 | Itanium C++ ABI | 与 GCC/Clang 兼容 |
| 调试信息 | DWARF | 标准调试格式 |

## 二、功能模块划分和优先级

### 2.1 模块架构

```
Driver（编译器驱动）
  +-- Basic（基础设施：SourceManager、Diagnostics、Language、Unicode）
  +-- Lex（词法分析+预处理：Lexer、Preprocessor、Token、双语关键字）
  +-- Parse（语法分析：递归下降解析器，表达式/语句/声明解析）
  +-- AST（AST 节点定义与内存管理：Expr/Stmt/Decl/Type 体系）
  +-- Sema（语义分析：类型检查、符号表、重载解析、模板实例化）
  +-- CodeGen（LLVM IR 生成：类型映射、表达式/语句/函数代码生成）
  +-- AI（AI 功能集成：多模型编排、智能诊断、优化建议）
  +-- Frontend（编译器实例：CompilerInstance、CompilerInvocation）
```

## 四、中英双语支持规划

### 4.1 双语支持架构

双语支持贯穿编译器所有阶段：

```
源代码（UTF-8，中英混合）
    |
Lexer（Unicode 标识符识别 + 中英文关键字映射）
    | Token 流（带语言标记）
Parser（双语语法支持，AST 节点记录源语言）
    | AST
Sema（双语诊断信息，Unicode 标识符符号表）
    | 类型化 AST
CodeGen（Unicode 标识符名称修饰，生成标准 C++ 兼容目标码）
```

| 双语预处理指令 | ✅ 完成 | 13 种映射（#包含/#定义/#如果 等） |

## 九、与行业标准的对比

### 9.1 与 Clang 的架构对比

| 组件 | Clang | BlockType | 一致性 | 差异说明 |
|------|-------|-----------|--------|---------|
| Lexer | ✅ | ✅ | 高 | BlockType 增加双语支持 |
| Preprocessor | ✅ | ✅ | 高 | BlockType 增加中文预处理指令 |
| Parser | ✅ | ✅ | 高 | 递归下降，与 Clang 一致 |
| Sema | ✅ | ✅ | 中 | BlockType Sema 曾为死代码，正在重构激活 |
| AST | ✅ | ✅ | 高 | 节点体系与 Clang 类似 |
| CodeGen | ✅ | ✅ | 高 | 架构一致，缺少 CGCall/CGExprComplex/CGExprScalar 分拆 |
| DebugInfo | ✅ | ✅ | 高 | DWARF 完整 |
| Mangler | ✅ | ✅ | 高 | Itanium ABI 一致 |
| Modules | ✅ | ⚠️ | 低 | BlockType 基础实现，Clang 完整 |
| Coroutines | ✅ | ⚠️ | 低 | BlockType 简化实现 |
| RTTI | ✅ | ⚠️ | 低 | BlockType 部分实现 |
| Templates | ✅ | ✅ | 中 | BlockType 完整但性能待优化 |

### 9.2 与 GCC/MSVC 的功能对比

| 特性 | GCC 14 | Clang 19 | MSVC 19.4 | BlockType |
|------|--------|----------|-----------|-----------|
| C++23 支持 | 部分 | 部分 | 部分 | 72% |
| C++26 反射 | ❌ | ❌ | ❌ | ⚠️ 基础 |
| C++26 Contracts | ❌ | 实验性 | ❌ | ⚠️ 部分 |
| 双语支持 | ❌ | ❌ | ❌ | ✅ 原生 |
| AI 集成 | ❌ | ❌ | ❌ | ✅ 原生 |

**BlockType 的差异化优势：**
1. **中英双语编程**：所有主流编译器中唯一原生支持
2. **AI 集成**：所有主流编译器中唯一原生集成
3. **C++26 反射**：与 Clang 同步的实验性实现

```
Source File (.cpp/.h)
    |
    v
+------------------+
|  SourceManager   |  文件读取、缓冲区管理
+--------+---------+
         |
         v
+------------------+
|    Lexer         |  词法分析 -> Token 流（双语支持）
+--------+---------+
         |
         v
+------------------+
|  Preprocessor    |  宏展开、#include、#embed
+--------+---------+
         |
         v
+------------------+
|    Parser        |  语法分析 -> AST（递归下降）
+--------+---------+
         |
         v
+------------------+
|     Sema         |  语义分析、类型检查、符号表
+--------+---------+
         |
         v
+------------------+
|    IRGen         |  LLVM IR 生成
+--------+---------+
         |
         v
+------------------+
|  LLVM Backend    |  优化、目标代码生成
+------------------+
```
