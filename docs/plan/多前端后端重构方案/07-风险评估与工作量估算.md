## 四、风险评估

### 4.1 编译/测试回归风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| IR转换引入语义差异 | 中 | 高 | 每个Task完成后对比BTIR→LLVM IR与现有CodeGen输出 |
| C++特有语义丢失(异常/RTTI/dynamic_cast) | 中 | 高 | Phase B中C++特有Task逐一验证 |
| 测试覆盖不足 | 低 | 中 | 每个Phase新增IR层单元测试+端到端测试 |
| IR指令遗漏导致某些C++特性无法表达 | 中 | 高 | 逐指令对照LLVM IR指令集；增量测试 |

### 4.2 性能影响评估（取自方案0量化数据）

| 方面 | 影响 | 分析 |
|------|------|------|
| 编译速度 | 轻微下降（~3-8%） | 引入IR中间层增加一次内存拷贝；IR→LLVM IR转换有开销 |
| 代码质量 | 无影响 | 最终仍使用LLVM优化管线，生成代码质量不变 |
| 内存使用 | 增加（~10-20%） | BTIR和LLVM IR同时存在于内存 |
| 二进制大小 | 增加（~5%） | IR库代码增加 |

**缓解**：Phase E 后可优化 IR→LLVM IR 转换为流式处理，减少内存峰值。

### 4.3 CGCXX 拆分风险（取自方案1专项）

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 拆分引入编译错误 | 低 | 低 | 先物理拆分（仅分文件），逻辑不变 |
| VTable生成迁移到后端 | 中 | 高 | 保留CGVTable.cpp在CodeGen中，Phase C迁移到LLVM后端 |
| 多重继承语义丢失 | 中 | 高 | 专项测试：多重继承、虚继承、thunk |
| 类型映射不一致 | 中 | 中 | IRTypeMapper和IRToLLVMConverter双向测试 |

### 4.4 与 Phase 8 工作的兼容性

| Phase 8 工作 | 兼容方案 |
|-------------|---------|
| TargetInfo/ABIInfo | TargetLayout镜像TargetInfo布局信息；ABIInfo移入后端层 |
| Mangler substitution | IRMangler复用substitution逻辑，不依赖CodeGenModule |
| Builtins | Builtins属于前端语义，在ASTToIRConverter中处理 |
| 调用约定 | 调用约定作为IR函数属性，LLVM后端映射到LLVM CallingConv |
| 优化pipeline | LLVM优化pipeline在LLVMBackend中复用，不受影响 |
| 目标特定优化 | 目标特定优化在LLVMBackend中执行，不受影响 |

### 4.5 其他风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 新旧路径并存导致维护负担 | 低 | 中 | 限制并存时间；尽快完成Phase E |
| IR设计不够通用，后续前端/后端扩展受限 | 低 | 高 | IR指令集参考LLVM IR和MLIR，保持通用性 |

| R-F06 | IR模糊测试发现大量现有bug，可能延迟Phase进度 | 中 | 中 | 开发节奏 | 模糊测试在Phase D引入，发现的bug按优先级处理；不阻塞Phase主线 |

### 5.2 执行顺序和并行策略

**严格串行**（因红线R3要求每阶段可编译可测试）：

```
Phase A (14.5天) → Phase B (29天) → Phase C (20天) → Phase D (7天) → Phase E (8天)
```

**Phase 内部并行**（取自方案0）：
- Phase A：A.1-A.4 可部分并行（类型/值/模块可同时开发）
- Phase B：B.1-B.2 可与 B.3 并行；B.4-B.6 必须串行
- Phase C：C.1 可与 C.2 并行；C.3-C.8 必须串行

**Phase B/C 并行机会**（取自方案0，可缩短总工期约5-7天）：
- Phase B 的 Task B.1-B.2（前端接口+基础转换）与 Phase C 的 Task C.1（后端接口）可并行
- Phase B 的 Task B.4-B.6（表达式/语句/C++ 特有）与 Phase C 的 Task C.2（IR→LLVM 转换）可部分并行

**考虑并行后总工期**：约 **71天**

### 5.3 关键路径

```
Phase A → Phase B (CGCXX 拆分) → Phase D → Phase E
```

CGCXX 拆分（Task B.7-B.8，5天）是整个项目的关键瓶颈。
