## 用户需求

开始 C++20 模块阶段2的开发工作，实现核心基础设施。

## 产品概述

为 BlockType 编译器实现 C++20 模块系统的核心基础设施，包括模块管理器、BMI 文件格式和模块依赖图构建。

## 核心功能

### 阶段2任务（预估13天）

1. **ModuleManager 接口设计**（3天）- 模块加载、缓存、生命周期管理
2. **BMI 文件格式实现**（5天）- Binary Module Interface 文件读写
3. **模块依赖图构建**（3天）- 拓扑排序、循环依赖检测
4. **模块编译流程集成**（2天）- 将模块编译集成到编译器主流程

### 已完成（阶段1）

- 符号表注册修复（ActOnModuleDecl/ImportDecl 调用 registerDecl）
- export 块解析支持（支持 `export { decl1; decl2; }`）
- 单元测试验证（5/5 测试通过）

## Implementation Approach

### 架构设计

采用分层架构，模块系统位于 Sema 和 Frontend 之间：

```
Frontend (CompilerInstance)
    ↓
Module (ModuleManager, BMI, DependencyGraph)  ← 新增
    ↓
Sema (符号表、作用域)
    ↓
AST (ModuleDecl, ImportDecl, ExportDecl)
```

### 核心组件设计

#### 1. ModuleManager

- **职责**: 模块加载、缓存、搜索路径管理
- **数据结构**: `llvm::StringMap<std::unique_ptr<ModuleInfo>>` 缓存已加载模块
- **关键方法**: `loadModule()`, `compileModule()`, `findModuleBMI()`

#### 2. BMI 文件格式

- **Header**: 64字节固定头部（魔数、版本、偏移量）
- **Section**: 模块元数据 → 依赖信息 → 导出符号 → AST 序列化
- **策略**: 阶段2采用简化序列化（仅元信息），阶段3评估完整 AST 序列化

#### 3. 模块依赖图

- **数据结构**: 有向图，`ModuleNode` 包含依赖和被依赖列表
- **算法**: DFS 拓扑排序、DFS 循环检测
- **输出**: 编译顺序、循环依赖链

### 与现有代码集成

- **Sema**: 新增 `ModuleManager *ModMgr` 成员
- **SymbolTable**: 扩展支持模块符号可见性标记
- **Parser**: 调用 ModuleManager 加载导入模块
- **Frontend**: 编译流程中集成模块编译步骤

## Implementation Notes

### 性能考虑

- 使用 `llvm::StringMap` 实现模块缓存，O(1) 查找
- BMI 文件使用内存映射（`llvm::MemoryBuffer`）加速读取
- 依赖图构建采用增量更新策略

### 错误处理

- 模块加载失败：记录诊断信息，返回 nullptr
- 循环依赖：检测并报告完整的循环链
- BMI 格式错误：验证魔数和版本号

### 向后兼容

- 不在模块中的代码保持原有行为
- ModuleManager 可选启用（通过编译选项）

## Key Code Structures

```cpp
// include/blocktype/Module/ModuleInfo.h
struct ModuleInfo {
  llvm::StringRef Name;
  llvm::StringRef Partition;
  std::string BMIPath;
  bool IsExported;
  bool IsPartition;
  llvm::SmallVector<llvm::StringRef, 8> Imports;
};

// include/blocktype/Module/BMIFormat.h
struct BMIHeader {
  char Magic[6];           // "BTBMI\0"
  uint32_t Version;
  uint32_t ModuleNameOff;
  uint32_t ModuleNameLen;
  uint32_t Flags;
  uint32_t NumImports;
  uint32_t NumExports;
  uint32_t ASTSectionOff;
  uint32_t ASTSectionSize;
  uint8_t Padding[28];
};
static_assert(sizeof(BMIHeader) == 64);

// include/blocktype/Module/ModuleManager.h
class ModuleManager {
  ASTContext &Context;
  DiagnosticsEngine &Diags;
  llvm::StringMap<std::unique_ptr<ModuleInfo>> LoadedModules;
  ModuleDecl *CurrentModule = nullptr;
  llvm::SmallVector<std::string, 4> SearchPaths;
public:
  ModuleInfo *loadModule(llvm::StringRef Name);
  void registerModuleDecl(ModuleDecl *MD);
  void setCurrentModule(ModuleDecl *MD);
  bool isModuleLoaded(llvm::StringRef Name) const;
};
```
